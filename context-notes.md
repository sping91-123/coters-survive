# 작업 맥락 메모

## 2026-05-13 6시간 반복 점검 4차.

- 매크로 레이더는 반복 방문 이유가 될 수 있지만, 자동 캘린더 연동 전에는 수동 데이터가 낡아 보이는 순간 신뢰가 크게 떨어진다.
- 사용자에게 ET와 KST를 섞어 보여주지 않는 기존 원칙은 유지하고, 내부에는 ISO 갱신 시각을 별도로 둔다.
- 화면에는 “최신 확인”, “갱신 확인 권장”, “갱신 필요”처럼 사용자가 바로 이해할 수 있는 신선도 문구만 노출한다.
- 운영 스모크는 매크로 갱신 시각 필드가 빠지는 회귀를 막는 역할로 둔다.

## 2026-05-13 6시간 반복 점검 3차.

- 글로벌 알림 페이지가 코인 알림과 분리된 뒤 기본 활성 알림이 너무 적게 보여서, 사용자가 글로벌 Pro의 가치를 바로 느끼기 어렵다고 판단했다.
- `stock-momentum`은 글로벌 시장 사용자가 기대하는 핵심 알림이므로 기본 활성화한다.
- 이 변경은 시장별 저장 키 분리 이후라 코인 알림 기본값에는 영향을 주지 않는다.
- 운영 스모크에 글로벌 모멘텀 알림 규칙을 추가해 다음 작업에서 실수로 빠지지 않게 한다.

## 2026-05-11 판독 UX 25개 항목 전면 정리 판단.

- 사용자가 제기한 25개 항목은 대부분 단순 취향 문제가 아니라 제품 신뢰와 초보자 이해도 문제로 판단했다.
- 매크로 일정은 혼선을 줄이기 위해 화면에는 한국시간만 표시한다. ET는 내부 참고값으로 남겨도 사용자 화면에는 노출하지 않는다.
- 청산 압력의 레버리지 밴드는 실제 CoinGlass식 청산 물량 지도가 아니라 현재가 기준 고배율 위험 거리 추정이다. 따라서 이름과 설명을 바꾸거나 메인 근거에서 낮춰야 한다.
- 종합, ICT, 기술지표는 철학이 다른 판독 방식이므로 한 화면에서 섞기보다 선택한 기준만 보이게 정리한다.
- 판독성향과 차트 오버레이는 서로 다른 설정이다. 판독성향은 계산값을 바꾸는 항목만 남기고, 오버레이는 차트 표시 설정으로 분리한다.

## 2026-05-11 출시 스모크 테스트 자동화 판단.

- 출시 직전에는 눈으로 모든 페이지를 여는 방식만으로는 빠뜨리는 경로가 생기기 쉽다.
- 핵심 공개 페이지, 약관, 개인정보, 환불 안내, 결제 진입 API를 한 번에 확인하는 로컬 스모크 테스트를 두면 배포 전 확인 속도가 빨라진다.
- 시장 데이터 API는 외부 거래소 응답에 영향을 받으므로 기본 스모크 테스트에서는 제외하고, 사용자 진입과 신뢰에 직접 필요한 경로를 우선 확인한다.

## 2026-05-07 분석앱 방향 전환

- 제품 톤은 타점 추천이나 자동 진입 후보가 아니라 코인 시장 구조 분석앱으로 맞춘다.
- 현재 베타는 코인 데이터를 우선 분석하고, 추후 해외선물까지 확대할 수 있다는 문구로 정리한다.
- ICT 구조가 중심이지만 RSI, MACD, 거래량, 볼린저밴드 등은 보조 도구로만 참고한다.
- PRO와 무료 판독권은 정식 상품 검증 전까지 숨기고, 베타 기간에는 기능을 전체 공개한다.
- 복기와 저장 기능은 베타 기간 중 정식 오픈 후 보존되지 않을 수 있음을 안내한다.
- 차트는 기본적으로 캔들 중심으로 보여주고, 구조 판독은 차트 아래 카드로 분리한다.
- 보수적 분석은 파란 계열, 공격적 분석은 빨간 계열로 구분한다.
- 공격적 분석은 정확도는 낮아질 수 있지만 완화된 조건으로 더 많은 후보를 보고 싶은 사용자를 위한 모드로 설명한다.

## 2026-05-08 상단 앱 메뉴 판단

- 정책 문서에서도 PRO 판매 전제를 제거해 사용자가 베타 공개형 제품으로 받아들이게 정리한다.
- 기존 방패형 아이콘은 제품명과 맞지 않으므로 레이더 링, 스윕, 블립을 가진 앱 아이콘으로 교체한다.

- 뉴스 API가 실제로 외부 수집에 실패하거나 환경변수 부족으로 비는 경우에도 사용자는 빈 페이지를 보면 “앱이 덜 됐다”고 느낀다. 최소한 오늘 확인할 매크로, 시장 영향, 관찰 포인트를 fallback 브리핑으로 남겨야 한다.
- 단순히 가짜 뉴스를 만드는 것은 신뢰를 해치므로, fallback은 공개 뉴스 원문을 꾸며내지 않고 “현재 수집 대기, 대신 오늘 확인할 레이더 포인트”로 분리한다.
- 재확인 결과 API는 실제로 뉴스 12개를 수집하고 있었고, 첫 브라우저 스냅샷은 로딩 전 화면이었다. 그래도 같은 오해가 다시 생기지 않도록 운영 스모크 테스트가 레이더뉴스 fallback, Groq 우선 호출, 코인 이슈 요약, 참고 뉴스 UI 계약을 확인하게 한다.
- 개발 서버의 오래된 chunk가 남으면 클라이언트 hydrate가 깨져 `useEffect`와 버튼 클릭이 동작하지 않을 수 있다. 뉴스 화면이 0개로 멈춰 보이면 먼저 브라우저 콘솔의 chunk 오류와 `.next` 캐시 상태를 확인한다.
- 로딩 중 숫자 카드가 0으로 보이면 “수집된 뉴스가 없다”는 인상을 주므로, 최초 로딩 상태는 “확인 중”으로 분리했다. API 응답 후에는 실제 수집 건수와 AI 브리핑이 정상 표시되는 것을 `/news` 브라우저 화면에서 확인했다.

# 2026-05-13 레이더뉴스 번역 품질 보강.

- 무료 번역 API는 빠르게 붙일 수 있지만 “인플레이션 인쇄물”, “불베어”처럼 금융 뉴스에서는 바로 어색하게 보이는 표현을 만들 수 있다. 유료 상품의 뉴스 화면에서는 이런 단어 하나가 신뢰를 크게 깎는다.
- 모든 뉴스를 사람이 직접 번역할 수는 없으므로, 반복 출현 가능성이 높은 헤드라인과 표현부터 고정 한국어 표현으로 보정한다. 특히 CPI, CFTC, bull-bear cycle, live markets류 제목은 시장 사용자가 바로 이해할 문장으로 바꾸는 편이 낫다.
- 후처리 규칙은 뉴스 원문을 꾸며내는 용도가 아니라 번역 품질을 다듬는 용도다. 방향성 분류와 출처 링크는 원문 기준을 유지한다.

# 2026-05-13 레이더뉴스 응답 속도 보강.

- 브라우저 기준으로 레이더뉴스가 완료되기까지 50초 이상 걸린 케이스가 있었다. 출시 앱에서 이 정도 대기는 “고급 AI 분석”이 아니라 “멈춘 화면”으로 느껴진다.
- 가장 큰 병목은 무료 외부 번역 API와 Groq 실패 후 Gemini까지 기다리는 긴 fallback 체인이다. 기본값은 빠른 로컬 제목 보정과 규칙 기반 fallback을 우선하고, 외부 무료 번역과 Gemini fallback은 환경변수로 명시적으로 켤 때만 사용하도록 나눈다.
- 뉴스 화면은 완벽한 장문보다 매일 빠르게 열리는 신뢰가 더 중요하다. AI가 늦으면 규칙 기반 브리핑으로 먼저 보여주고, 사용자가 핵심 뉴스와 시장 영향만 바로 확인할 수 있어야 한다.

# 2026-05-13 레이더뉴스 브리핑 표시 정리.

- 내부적으로 AI가 성공했는지, 규칙 기반 fallback인지 구분하는 것은 운영에는 필요하지만, 사용자 화면에서 “규칙 기반 브리핑”이 크게 보이면 상품감이 약해진다.
- 사용자에게 중요한 것은 모델명이 아니라 오늘 확인해야 할 뉴스, 시장 영향, 행동 기준이다. 그래서 브리핑 배지는 “레이더 종합 브리핑”으로 통일하고 내부 model 값은 데이터에만 남긴다.

# 2026-05-13 알림 규칙 시장별 저장 판단.

- 알림 모니터 자체는 코인과 글로벌을 모두 감시하도록 보강되어 있지만, 사용자가 켜둔 알림 규칙은 하나의 localStorage 키를 공유하고 있었다.
- 코인과 글로벌을 별도 앱처럼 운영하려면 알림 규칙도 시장별로 분리되어야 한다. 다만 기존 사용자가 켜둔 코인 알림 설정은 잃지 않도록 코인 화면에서는 이전 공통 키를 fallback으로 읽는다.

# 2026-05-13 정식 출시 문구 정리.

- 사용자는 베타가 아니라 정식 출시 제품처럼 보이기를 원한다. 그래서 사용량, 알림, 복기 화면에서 “준비”, “출시 후”, “보존되지 않을 수 있음”처럼 미완성으로 읽히는 문구를 줄인다.
- 실제 서버 권한과 푸시 인프라는 아직 더 보강해야 하지만, 사용자 화면에서는 개발 계획을 설명하기보다 현재 쓸 수 있는 기능과 계정 기준 저장 흐름을 말하는 편이 결제 전환에 더 맞다.
- 복기장은 무조건 보존을 약속하기보다 로그인 전 기록은 브라우저 한정, 로그인 후 기록은 계정 기준 저장이라는 사실을 짧게 안내한다.
