# 작업 맥락 메모

## 2026-05-07 분석앱 방향 전환

- 제품 톤은 타점 추천이나 자동 진입 후보가 아니라 코인 시장 구조 분석앱으로 맞춘다.
- 현재 베타는 코인 데이터를 우선 분석하고, 추후 해외선물까지 확대할 수 있다는 문구로 정리한다.
- ICT 구조가 중심이지만 RSI, MACD, 거래량, 볼린저밴드 등은 보조 도구로만 참고한다.
- PRO와 무료 판독권은 정식 상품 검증 전까지 숨기고, 베타 기간에는 기능을 전체 공개한다.
- 복기와 저장 기능은 베타 기간 중 정식 오픈 후 보존되지 않을 수 있음을 안내한다.
- 차트는 기본적으로 캔들 중심으로 보여주고, 구조 판독은 차트 아래 카드로 분리한다.
- 보수적 분석은 파란 계열, 공격적 분석은 빨간 계열로 구분한다.
- 공격적 분석은 정확도는 낮아질 수 있지만 완화된 조건으로 더 많은 후보를 보고 싶은 사용자를 위한 모드로 설명한다.

## 2026-05-07 AI 종합 피드백 추가

- 기존 AI 코멘트는 셋업 카드 1장용이라 차트 판독 전체를 설명하기에는 범위가 좁다.
- 차트 판독 화면에는 선택 코인, 선택 타임프레임, 종합 점수, TF별 MSB/CHoCH, OB/FVG/POC/PD, 보조지표, 리스크 플래그를 묶은 긴 문장형 피드백을 별도 API로 제공한다.
- AI 응답은 매수·매도 지시가 아니라 시장 구조 해석, 우세한 방향, 조심할 지점, 다음 확인 조건을 설명하는 교육용 문장으로 제한한다.
- Gemini 호출 실패나 키 미설정 시에도 규칙 기반 폴백 문장을 보여줘 화면이 비지 않게 한다.

## 2026-05-08 출시 전 코드리뷰 보완

- 전체 코드리뷰에서 가장 먼저 막아야 할 지점은 AI API 비용 노출, 스캐너 API 과호출, 복기 서버 저장 누락으로 정리했다.
- 베타 기간에는 관심코인과 주요 기능을 멤버십 게이트 없이 열어두고, 결제 전 사용자 반응과 유지율을 먼저 확인한다.
- 인증은 현재 REST와 localStorage 기반을 유지하되, 만료된 access token은 refresh token으로 갱신해 갑작스러운 로그아웃을 줄인다.
- 복기 테이블은 스캐너 저장과 결과 추적을 제품 핵심 기능으로 보기 때문에 `scout`, `outcome`, `outcome_at`, `scout_snapshot`을 저장 대상으로 확장한다.
- `ChartScanner`는 직관적이지만 너무 일반명사라 보류했고, 이후 제품명은 `차트 레이더 / Chart Radar`로 정리했다.

## 2026-05-08 차트 레이더 브랜딩 결정

- 한글 제품명은 `차트 레이더`, 영문 제품명은 `Chart Radar`로 정한다.
- 사용자에게는 “레이더 돌려봤어?”, “진입 전 레이더 한 번 찍어봐”처럼 말할 수 있는 브랜드 언어를 만든다.
- UI에서는 스캐너보다 레이더, 감지, 브리핑, 구조 신호 같은 표현을 우선 사용한다.
- 레이더 시각 요소는 첫 단계에서 CSS 기반 원형 마크와 회전 스윕 애니메이션으로 구현한다.

## 2026-05-08 베타 UI 단순화

- 아직 결제를 받지 않으므로 환불 정책과 PRO 결제 관련 노출은 숨긴다.
- 로그인은 구글만 사용하고, 카카오 사업자 인증 예정 문구는 제거한다.
- 단타, 스캘핑, 스윙, 데이처럼 성과를 약속하는 모드 대신 5m, 15m, 1h, 4h, 1d 타임프레임별 레이더 관찰로 보여준다.
- AI 브리핑 로딩은 단순 스피너가 아니라 화면 중앙의 레이더 스캔 오버레이로 보여준다.

## 2026-05-08 후보 레이더 랭킹 단순화

- 차트 레이더 후보 패널은 특정 타임프레임 필터보다 전체 후보 중 급과 점수가 높은 순서를 우선 보여준다.
- 보수적 분석과 공격적 분석은 서로 떨어진 옵션이 아니라 하나의 세그먼트 선택처럼 붙여서 비교 가능하게 둔다.
- 타임프레임 선택은 차트 판독 영역의 역할로 남기고, 후보 패널은 5m부터 1d까지 통합 레이더 결과를 보여준다.
- 레이더 마크는 단순 원형 아이콘보다 다중 링, 스윕, 블립 점이 있는 장치 느낌으로 강화한다.

## 2026-05-08 레이더 UI 가독성 판단
- 현재 화면은 후보 카드마다 설명, 경고, AI 코멘트가 반복되어 초보 사용자에게 읽을 부담이 크다.
- 카드 안에서는 현재가, 검토 구간, 리스크 기준, 참고 목표, 구조 신뢰도만 먼저 보이게 하고 긴 교육 문구는 섹션 상단과 하단 안내로 모은다.
- 준비도는 무엇을 준비한다는 뜻인지 불명확하므로 후보 구조의 질을 뜻하는 구조 신뢰도로 바꾼다.
- 진입진단과 계산기는 기능이 겹쳐 보이므로 메뉴에서 원칙 점검과 수량 계산으로 분리해 역할을 먼저 이해시키는 방향으로 간다.

## 2026-05-08 플루토식 구조를 차트 레이더식으로 변환하는 판단
- 플루토의 장점은 기술지표 자체가 아니라 모바일에서 계속 스크롤하며 볼 수 있는 카드 리듬이다.
- 차트 레이더는 RSI·MACD 중심 앱이 아니라 MSB, CHoCH, OB, FVG, Sweep, OTE, PD, POC 중심의 ICT 구조 판독 앱으로 남긴다.
- 상단에는 기본코인, 관심코인, 레이더 대시보드, AI 브리핑, 프리미엄에 해당하는 제품 흐름을 보여준다.
- 5m, 15m, 1h, 4h, 1d는 모바일 하단 고정 메뉴로 빼서 어떤 타임프레임을 보고 있는지 즉시 바꿀 수 있게 한다.
- 첫 화면에서 긴 설명보다 현재 결론, 방향 근거 비율, AI 한줄 요약을 먼저 보여준다.

## 2026-05-08 베타 첫인상 강화 판단
- 사용자가 느끼는 답답함은 UI 밀도만의 문제가 아니라, 제품이 처음 10초 안에 “오늘 뭘 봐야 하는지”를 선명하게 말하지 못하는 데서 온다.
- 결제 전 베타에서는 프리미엄 메뉴를 전면에 둘 이유가 없으므로 제거하고, 전체 공개형 리서치 도구처럼 보이게 한다.
- 첫 화면은 지표 목록보다 핵심, 위험, 다음 확인 조건을 묶은 오늘의 레이더 브리프가 먼저 와야 한다.
- 후보 패널은 타점 리딩처럼 보이면 신뢰 리스크가 커지므로 시장 레이더 TOP 감지와 관찰 리포트 톤으로 낮춘다.

## 2026-05-08 시장 레이더 지휘판 판단
- 베타에서 “와 대박” 반응을 만들려면 차트 하나를 고르게 하기 전에 앱이 먼저 시장 전체를 훑었다는 느낌이 필요하다.
- 기존 `SetupScoutPanel`은 상세 후보 목록 역할로 남기고, 첫 화면에는 더 짧고 강한 시장 레이더 지휘판을 추가한다.
- 지휘판은 강한 감지 수, 관찰 수, 롱/숏 쏠림, TOP 코인을 요약해 사용자가 앱을 켜는 이유를 즉시 만든다.
- 표현은 타점 추천이 아니라 “오늘 먼저 확인할 시장 구조”로 제한해 신뢰 리스크를 낮춘다.

## 2026-05-08 레이더 노트 판단
- 사용자가 플루토식으로 좋게 느낀 부분은 차트보다 “지금 궁금할 질문을 앱이 먼저 정리해주는 흐름”이다.
- 외부 뉴스나 수익 예측을 억지로 붙이지 않고, 시장 보드와 레이더 감지 데이터만으로 오늘의 시장 온도, 거래대금 1위, 레이더 최상단을 정리한다.
- 질문 카드는 AI 챗봇처럼 보이되 실제로는 룰 기반 짧은 답변을 제공해 비용과 환각 위험을 낮춘다.
- 이 패널은 유료 전환 전에도 “앱이 나 대신 시장을 훑었다”는 첫인상을 만들기 위한 베타 핵심 장치로 둔다.

## 2026-05-08 상단 앱 메뉴 판단
- 사용자가 참고한 앱의 강점은 메뉴가 첫 화면에 바로 보여서 서비스 구조를 즉시 이해시키는 점이다.
- 기존 메뉴는 코인 차트 카드 안에 있어 첫 화면에서는 제품 흐름이 늦게 드러났다.
- 상단 메뉴를 별도 컴포넌트로 분리하고, 차트 카드 안 중복 메뉴를 제거해 기본코인 화면을 더 가볍게 만든다.
- 프리미엄은 아직 숨기는 방향이므로 기본코인, 관심코인, 대시보드, AI 브리핑만 노출한다.
- 정책 문서에서도 PRO 판매 전제를 제거해 사용자가 베타 공개형 제품으로 받아들이게 정리한다.
- 기존 방패형 아이콘은 제품명과 맞지 않으므로 레이더 링, 스윕, 블립을 가진 앱 아이콘으로 교체한다.
- 검색 메타데이터도 선물 전용이 아니라 코인 분석 앱으로 보이게 맞춘다.

## 2026-05-08 판독 철학 분리 결정.
- 차트 레이더는 ICT 구조를 중심으로 시작했지만, 모든 사용자가 ICT를 쓰는 것은 아니므로 종합, ICT 구조, 기술지표 기준을 명확히 분리한다.
- 기술지표 기준은 RSI와 MACD 몇 개만 보조로 붙이는 화면이 아니라 별도 레이더 화면으로 만든다. MA, EMA, MACD, ADX, RSI, Stochastic, CCI, MFI, ATR, Bollinger, OBV, VMA, VWAP, CMF, 피보나치, 캔들 패턴을 한 번에 보되 매수·매도 신호처럼 표현하지 않는다.
- 공포·탐욕은 공식 외부 지수가 아니라 현재 선택 코인의 캔들 기반 심리 참고값으로 표시한다. 공식 지수 연동은 별도 API 안정성 검토 후 진행한다.
- ZigZag 5는 기존 Pine 지표와의 일치율을 위해 유지한다. 다만 5분에서는 노이즈가 늘 수 있고 4시간 이상에서는 큰 스윙 기준이 더 나을 수 있으므로 이후 3, 5, 8 또는 ATR 적응형 민감도 검증을 진행한다.

## 2026-05-08 모바일 조작 동선 결정.
- 차트 레이더는 코인과 타임프레임, 판독 기준을 자주 바꾸며 보는 앱이므로 모바일에서 상단 선택 UI는 불편하다.
- 모바일에서는 하단 고정 패널에 코인, 종합/ICT/기술지표, 5m~1d, 분석/요약/근거 버튼을 묶고 데스크톱에서는 기존 상단 선택을 유지한다.

## 2026-05-08 데스크톱 조작 위치 통일.
- 사용자가 컴퓨터에서도 코인, 타임프레임, 판독 기준을 바꿀 때 위아래 이동이 번거롭다고 봤다.
- 상단 선택 UI는 숨기고 하단 고정 패널을 모든 화면 크기에서 사용하게 바꿨다. 데스크톱에서는 그리드로 압축해 차트 영역을 덜 가리게 했다.

## 2026-05-08 관심코인 토글 UX.
- 사용자는 관심코인 추가 모달에서 이미 선택된 코인을 다시 누르면 제거될 것으로 기대한다.
- 기존 구현은 이미 추가된 항목 클릭을 무시해서 "추가는 되는데 빠지지 않는" 느낌을 줬다.
- 모달 선택 버튼을 추가와 제거를 모두 처리하는 토글로 바꾸고, 기존 칩의 X 제거 동작도 그대로 둔다.
- 하단 코인/타임프레임/레이더 선택 패널은 데스크톱에서도 고정으로 보이므로 survival 페이지 하단 여백을 함께 늘렸다.
- 레이더 판독 저장과 후보 저장은 로그인 세션을 직접 읽고 있었기 때문에 access token 만료 시 저장 실패로 이어질 수 있었다.
- 공통 `getActiveSupabaseSession`으로 저장 직전 세션을 갱신하고, 후보 저장은 서버 저장 실패 시 브라우저 복기로 대체 저장한다.

## 2026-05-08 하단 패널 단순화.
- 사용자는 하단의 분석, 요약, 근거 버튼 중 요약과 근거를 빼고 더 가볍게 쓰길 원했다.
- 코인 선택은 BTC와 ETH만 직접 노출하고 나머지는 "그 외" 팝업에 넣어 하단 패널의 가로 혼잡도를 줄였다.
- "그 외" 안의 코인을 선택하면 팝업은 닫히고, 선택된 알트 코인은 그 외 버튼의 라벨로 표시된다.

## 2026-05-08 TOP 레이더 중복 제거.
- 사용자는 BTC 일봉, BTC 5분, BTC 15분처럼 같은 코인이 TOP를 모두 차지하면 의미가 없다고 지적했다.
- 서버는 모드별 중복 제거를 하지만, 화면에서 단타와 스윙 결과를 합친 뒤 다시 정렬하면서 같은 코인이 중복될 수 있었다.
- 시장 레이더 TOP, 오늘의 레이더 노트, 후보 패널은 합산 결과에서 코인당 최고 후보 하나만 노출하도록 바꿨다.
## 2026-05-08 레이더뉴스 방향.
- 코인니스 API 제휴는 매출이 생긴 뒤 진행하는 장기 과제로 둔다.
- 지금은 코인니스 원문을 자동으로 긁거나 AI로 다시 써서 게시하지 않는다. 약관상 원문 재가공·재게시·스크래핑 리스크가 있기 때문이다.
- MVP는 공개 RSS가 허용된 해외 뉴스 소스의 제목, 링크, 시간만 가져오고 본문은 저장하지 않는다.
- 레이더뉴스의 차별점은 기사 복제가 아니라 제목에서 관련 코인, 방향성 압력, 긴급도, 확인할 리스크를 차트 레이더식으로 분류하는 것이다.
- 코인니스나 텔레그램 속보는 사용자가 직접 문장을 붙여넣어 개인용으로 해석하는 기능으로 먼저 제공한다.
- 상단 메뉴는 제품 페이지 단위로 분리하는 방향을 유지한다. BTC / ETH, 알트코인, 레이더뉴스, 매매복기, 계산기가 현재 1차 구조다.

## 2026-05-08 페이지 범위 정리.
- BTC / ETH는 메이저 코인 전용 화면으로 정리했다. 알트 관찰, 시장 보드, 전체 후보 스캐너는 이 화면에서 빼서 사용자가 비트와 이더 흐름에만 집중하도록 했다.
- 알트코인 화면은 BTC와 ETH를 제외한 관심코인과 후보 레이더를 다루도록 분리했다.
- 레이더뉴스는 해외 RSS 제목을 한국어로 우선 보여주고, 원문 제목은 작은 보조 텍스트로 남기는 방향으로 정리했다.
- 매매복기와 계산기는 기존 탭 메뉴 대신 Chart Radar 상단 메뉴를 사용해서 다른 주요 페이지와 진입 경험을 맞췄다.

## 2026-05-09 레이더뉴스 방향 수정.
- 사용자는 단순 기사 목록이 도움이 약하다고 판단했다. 뉴스 페이지의 가치는 기사 자체가 아니라 “오늘 시장에서 무엇을 신경 써야 하는지”를 정리해주는 데 있다.
- 공개 RSS는 출처와 원문 링크를 확인하는 재료로만 쓰고, 화면의 중심은 AI 종합 브리핑으로 둔다.
- 브리핑은 주요 이슈, 시장 영향, 투자 판단 포인트, 마지막 뉴스 정리로 구성한다. 직접적인 매수·매도 신호나 수익 보장은 피하고, 교육용 시장 해석과 리스크 점검 관점으로 표현한다.
- AI API가 느리거나 실패할 수 있으므로 규칙 기반 대체 브리핑을 반드시 유지한다.

## 2026-05-09 Groq 뉴스 브리핑 연결.
- 사용자가 Groq API 키를 `.env.local`에 추가했으므로 레이더뉴스 생성 순서를 Groq 우선으로 바꾼다.
- Groq는 서버 라우트에서만 호출하고 브라우저에는 키를 노출하지 않는다.
- 운영 안정성을 위해 Groq 실패나 한도 초과 시 Gemini로 넘기고, Gemini도 실패하면 기존 규칙 기반 브리핑을 보여준다.
- 기본 모델은 `qwen/qwen3-32b`로 두되, `.env.local`의 `GROQ_MODEL`로 바꿀 수 있게 한다.

## 2026-05-09 매크로 전광판과 뉴스 보완.
- 뉴스 페이지는 단순 기사 목록보다 "오늘 무엇을 조심해야 하는지"가 먼저 보여야 가치가 커진다.
- 최상단에 CPI, PPI, FOMC, 고용, 실업수당, 미 10년물, DXY를 흐르는 전광판으로 배치한다.
- 아직 경제 캘린더 API를 연결하지 않았으므로 정확한 발표 시각과 실제 수치는 별도 확인 문구를 함께 둔다.
- 뉴스 브리핑은 긴 문장만 두지 않고 뉴스 톤과 가장 먼저 볼 이슈를 상단에서 짧게 보여준다.

## 2026-05-09 공개 베타 완성도 강화.
- 사용자는 앱스토어에 올라가도 이상하지 않을 정도의 완성감을 원한다. 당장 가능한 최우선 개선은 기능 추가보다 매일 켜야 할 첫 화면 이유와 전역 테마 완성도다.
- 첫 화면에는 이미 만들어져 있던 시장 레이더 지휘판과 오늘의 레이더 노트를 다시 배치해, 차트를 보기 전 시장 전체를 훑은 느낌을 먼저 준다.
- 라이트/다크 모드는 별도 제품 경험으로 느껴지도록 전역 토글을 추가하고, 기존 다크 중심 색상은 라이트 모드에서 읽히도록 CSS 보정으로 처리한다.
